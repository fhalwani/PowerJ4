Auditors

Auditing allows you to review how PowerJ is interpreting and manipulating the data and make sure it is done correctly. Errors can occur due to several reasons:


  • Programatic error: programmers are humans and will make mistakes.

  • Ambiguity: programmers cannot predict and test for all possible scenarios; for example, we don't know how the program will treat a prostatectomy that has both an adenocarcinoma and a lymphoma combined in the same case.

  • Information error: a specimen, order or task that has a 10 unit value but has been mistakenly entered as 1.0 in its template.

  • Upstream error: A specimen has been inappropriately entered in the AP system. A common example is a medical liver biopsy is entered as a mass lesion liver biopsy.


PowerJ offers 3 modules to audit and review cases, and 1 module to correct specimen errors in the upstream AP system. These modules are:


  • Finals: cases reported and finalized showing workflow and workload units.

  • Pending: cases in the grossing room, histology or pathologists' desk showing workflow and predicted workload.

  • Errors: Cases detected by PowerJ and tagged as having an error; the most common error is having inadequate information about the specimen/test category, or encountering a new unknow specimen/test.

  • Specimen Editor: A screen allowing you to correct the information in your institution system without having to amend the case.
