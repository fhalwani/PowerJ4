Setup

This screen is used to change the variables required by PowerJ. It is divided into 3 groups:


  • Server: the variables required to login to your institution pathology system (PowerPath, Copath):

    • Server Name: the url for the database server

    • Database Name: the name of the database in the server

    • Port no: the TCP port to use to connect to the server

    • Login Name: the user name that PowerJ uses to login automatically to the server

    • Password: the password of the above user name that PowerJ uses

  • Workflow: the variables required for backlog and turnaround time analysis:

    • Update every: the frequency of workflow daily updates in minutes (range 10-120 minutes)1

    • Wakeup every: the frequency the computer clock is checked (range 1-15 minutes)1

    • Opening Hour: the hour to start counting business hours

    • Closing Hour: the hour to end counting business hours

    • Saturday closed: is Saturday a work day or holiday?

    • Sunday closed: is Sunday a work day or holiday?

    • Routing cutoff: The cutoff time for routing a case to be considered today vs tomorrow's work

  • Workload: the variables required for the workload analysis

    • Coder 1-4

      • Name: CAP vs L4E

      • Active: to include or exclude from the workload calculations

      • FTE: Total annual points for one full time equivalent

    • Minimum date: earliest date to calculate workload

    • Business days: total number of workdays per year


    It is recommended that if you are using the desktop version of PowerJ that you set these 2 variables at maximum range. But if you are using the client/server version of PowerJ with a dedicated powerful server to set it at the minimum range.
